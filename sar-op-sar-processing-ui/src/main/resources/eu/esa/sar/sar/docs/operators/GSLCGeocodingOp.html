GSLC Orthorectification

GSLC Terrain Correction

Description

The GSLC Terrain Correction operator performs Geocoded Single Look Complex (GSLC) generation by transforming SAR data from slant-range geometry to a map projection while preserving phase information. This process is essential for applications that require analysis of the complex SAR signal in a geographic coordinate system, such as interferometry (InSAR) and polarimetry (PolSAR).

Technical Background

Traditional SAR geocoding methods focus on correcting geometric distortions in the magnitude of the SAR image, often discarding the phase. However, the phase component of the complex SAR signal contains valuable information about the sensor-to-target distance. Preserving this phase during geocoding is crucial for advanced applications.

The GSLC generation process involves the following key steps:

  1. Precise Geometry Calculation: Using precise orbit state vectors and a Digital Elevation Model (DEM), the operator accurately determines the geographic coordinates (latitude, longitude, height) for each pixel in the output grid.
  2. Zero Doppler Search: For each point on the ground, the operator finds the corresponding azimuth time (zero Doppler time) in the source SAR image.
  3. Slant Range Calculation: The precise distance (slant range) from the sensor to the ground point is calculated. Optional corrections for Solid Earth Tide, tropospheric path delay, and ionospheric path delay may be applied at this step.
  4. Phase Flattening (Carrier Phase Compensation): Before resampling the complex data, the operator simulates the phase contribution from the sensor-to-target path geometry (the "carrier" phase). This simulated phase is then removed from the original complex signal. This "flattens" the phase, making the subsequent resampling process more accurate as it deals with a signal with lower fringe frequencies. The flattened complex value C' is computed as:
    C' = C * exp(-j * φsim)
    where C is the original complex value and φsim is the simulated phase:
    φsim = 4π * R / λ
    (R is the slant range, λ is the wavelength).
  5. High-Fidelity Complex Resampling: The flattened complex (I and Q) bands are resampled from the source slant-range grid to the target map grid using a high-fidelity interpolator (e.g., a truncated Sinc function) to preserve signal integrity.
  6. Phase Restoration (default): By default (Output flattened complex data = false), the simulated phase is re-introduced to the resampled complex signal. This restores the absolute phase relative to the new map geometry. The final complex value Cgeo is:
    Cgeo = C'resampled * exp(+j * φsim)
    This restored-phase output is what InSAR requires — the master · conj(slave) product then carries the differential phase (4π/λ)(RM − RS).

This process results in a Single Look Complex (SLC) image that is precisely geocoded to a map projection, with its phase correctly representing the sensor-to-target path length for each pixel in the new grid.

Output metadata stamps

The operator writes two custom attributes into the target product's abstracted metadata so downstream operators (notably CreateStack) can rebuild the same processing context when given only the GSLC on disk:

Attribute Type Purpose
gslc_source_slc_path string (file path) Absolute path of the slant-range SLC this GSLC was geocoded from. CreateStack reads this stamp when it needs to cross-correlate the master against a raw slave SLC to estimate the coregistration bias. If the stamp is missing (e.g. the master is a subset of a GSLC and the subset operation stripped it), CreateStack falls back to a name-based search for the source SLC in the master's directory.
gslc_output_flattened string ("true" / "false") Whether the GSLC was built with Output phase-flattened complex data = true. Used by CreateStack to ensure an auto-built slave GSLC matches the master's flattening state. Mixing flattened/non-flattened bands in the same stack produces a meaningless interferometric phase.

Processing Workflows

Single GSLC Generation

Stripmap SLC

  S-1 SM / ENVISAT ASAR IMS SLC Product
    → Apply-Orbit-File
    → (optional) Thermal-Noise-Removal
    → GSLC-Terrain-Correction

TOPS (IW/EW) SLC

  S-1 IW SLC Product
    → Apply-Orbit-File
    → TOPSAR-Split (select one subswath, e.g. IW2, and polarisation)
    → GSLC-Terrain-Correction

Repeat for each subswath if full-swath coverage is needed. The operator handles burst deramping, geocoding, and burst mosaicking internally.

GSLC-Based InSAR

The GSLC approach replaces the explicit BackGeocoding / ESD / Deburst / Merge / TerrainCorrection chain with a single geocoding step per acquisition. Each scene becomes a phase-preserving complex product on a common map grid, and the rest of the InSAR chain consumes those directly.

Simplest path — only the master is geocoded explicitly

  Master SLC (Date 1)            Secondary SLC (Date 2)
    → Apply-Orbit-File          → Apply-Orbit-File
    → GSLC-Terrain-Correction   (raw SLC, no GSLC step)
      ↓ master GSLC                       ↓
            CreateStack(master GSLC + raw secondary SLC)
                       ↓
            (CreateStack auto-geocodes the secondary on the master's grid,
             reloads the master's source SLC from the gslc_source_slc_path stamp,
             cross-correlates to estimate the residual (Δr, Δa) bias,
             rebuilds the secondary GSLC with that bias, and stacks the two.)
                       ↓
            Interferogram → (optional) Goldstein-Phase-Filtering → Phase-Unwrapping

Explicit path — both scenes geocoded up front

  Master SLC                            Secondary SLC
    → Apply-Orbit-File                  → Apply-Orbit-File
    → GSLC-Terrain-Correction           → GSLC-Terrain-Correction
      ↓ master GSLC                         ↓ secondary GSLC
                CreateStack
                     ↓
            Interferogram → ...

In this path you generally also want to run cross-correlation to find the sub-pixel residual offset and feed it back as Slant-range offset (pixels) / Azimuth offset (pixels) on the secondary's GSLC step. The simplest path above does this automatically.

Advantages of the GSLC approach:

  • Fewer processing steps (no deburst, merge, or separate terrain correction)
  • Each acquisition can be processed independently and in parallel
  • The output is directly geocoded — ready for time-series analysis
  • Consistent geocoding across all acquisitions (same DEM and geometry)
  • Secondary coregistration is folded into CreateStack — user runs GSLC once on the master, feeds raw SLC secondaries, and CreateStack handles the rest.

Multi-Subswath Coverage

For full IW coverage (IW1 + IW2 + IW3), process each subswath independently:

  For each subswath (IW1, IW2, IW3):
    TOPSAR-Split → GSLC-Terrain-Correction

  Then mosaic the 3 GSLC subswaths per date.

Alternatively, process each subswath interferogram independently, then mosaic the interferograms.

Technical Notes

TOPS Azimuth Deramping

In TOPS (IW/EW) mode, the antenna steers during each burst, imparting a quadratic phase ramp in azimuth. This ramp must be removed (deramped) before any interpolation to prevent aliasing. The operator handles this internally for TOPS products:

  1. For each burst, the azimuth deramp and demodulation phase is computed using the Doppler rate and centroid annotations from the product metadata.
  2. The phase is applied to the source complex data before resampling.
  3. After resampling, the interpolated phase is used to re-ramp the data.
  4. Range phase flattening is then applied as in the Stripmap case.

Stripmap Azimuth Deramp (residual Doppler centroid)

For Stripmap products with a non-zero residual Doppler centroid (e.g. ENVISAT ASAR IMS), the operator applies a sample-by-sample azimuth deramp / reramp using the Doppler_Centroid_Coefficients metadata field. This prevents phase aliasing during complex interpolation and is essential for coherent GSLC stacking.

Phase Flattening

The operator removes the geometric (carrier) phase before resampling to reduce fringe density and improve interpolation accuracy. The simulated phase is:

φsim = 4π R / λ

By default (Output phase-flattened complex data = false), the simulated phase is restored after resampling, so the output complex band carries the natural SLC carrier. This is what InSAR needs: a master/slave pair's conjugate product then yields the differential phase (4π/λ)(Rmaster − Rslave).

When the parameter is set to true, the simulated phase is not restored — the output holds only the local scattering coefficient. That is useful for non-InSAR analysis (e.g. coherent magnitude statistics, PolSAR decomposition on a single date) but produces a noise interferogram if such a GSLC is paired with a non-flattened counterpart. Use one mode consistently across all dates in a stack.

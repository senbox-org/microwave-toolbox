SBAS Inversion

SBAS Inversion Operator

The SBAS Inversion operator inverts a stack of unwrapped small-baseline interferograms for per-epoch phase, displacement, and a weighted linear-velocity model, following Berardino, Fornaro, Lanari & Sansosti (IEEE TGRS 2002).

Per pixel it builds the design matrix A of size M× (N-1) from the network of pairs, drops equations whose coherence is below the threshold, and solves the weighted normal equations x = (ATWA)-1 ATWφ. Tikhonov regularisation activates automatically when the matrix is ill-conditioned. Per-epoch phase is then converted to LOS displacement (d = −λ / 4π · φ) and fit to a linear-in-time model for velocity.

Input

  • A multi-temporal stack containing, per pair k:
    • Unwrapped phase band, Unit.PHASE or Unit.ABS_PHASE. Naming convention as emitted by Snaphu Import: Unw_Phase_ifg_<...>_<masterDate>_<slaveDate>.
    • Coherence band, Unit.COHERENCE, e.g. coh_<swath>_<POL>_<masterDate>_<slaveDate>.
  • Acquisition dates are extracted as the last two ddMMMyyyy tokens in each band name.

SBAS Inversion expects the Unw_Phase_ifg_* naming emitted by Snaphu Import and pairs each with the matching coh_* band by trailing date pair. Phase Linking (step 1) is optional — SBAS works on plain pairwise unwrapped interferograms too.

Output

  • phase_<date> — per-epoch unwrapped phase relative to the reference epoch (rad).
  • displacement_<date> — LOS displacement (mm).
  • velocity — weighted linear fit (mm/year).
  • velocity_uncertainty — 1-sigma velocity (mm/year).
  • temporal_coherence — residual-based quality, [0, 1].
  • residual_<m>_<s> — per-pair residual (optional).
  • closure_phase_rms — RMS triplet closure phase (optional).

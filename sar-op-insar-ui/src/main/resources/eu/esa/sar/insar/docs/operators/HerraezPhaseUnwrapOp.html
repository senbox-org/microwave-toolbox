Herraez Phase Unwrapping Operator

In-process 2D phase unwrapping using the quality-guided path-following algorithm of Herráez, Burton, Lalor & Gdeisat (2002). Designed as an alternative to snaphu for small-to-medium interferograms where launching a native helper is impractical (e.g. batch graphs, headless workflows, modest scene size).

Algorithm

  1. Reliability map. Per pixel, compute the second-difference reliability over 3×3 wrapped-phase neighbours (Herráez eq. 2). High reliability = small local phase variation = likely unambiguous unwrap.
  2. Edge construction. Each 4-connected pair becomes an edge whose weight is the sum of its endpoints' reliabilities.
  3. Sort & merge. Edges sorted descending by weight. Walk in order; for each edge, union-find merges the two groups while adjusting the slave group's wrapped-cycle offset so the cross-edge phase difference is in (-π, π].
  4. Output. Per pixel the absolute unwrapped phase = wrapped phase + 2π · cycleOffset.

The scene-global single pass runs once in initialize() and the result is cached so each tile only reads it. This is well-suited to typical InSAR products up to ~5000×5000 px on a desktop. For larger scenes, prefer snaphu's MCF/MST solvers.

Input

  • Every wrapped interferometric phase band in the source product (unit = phase or abs_phase, radians in [-π,π]) is unwrapped. No band selector is required.
  • Each phase band is auto-paired with a coherence band by matching the master/slave name suffix. If a match is found and Coherence Min > 0, pixels below the threshold are masked from unwrapping. If no coherence band is found, the phase band is unwrapped without coherence masking.

Output

  • One Unw_Phase_* band per input phase band - absolute unwrapped phase in radians (unit = abs_phase).

Parameters

ParameterDefaultNotes
Minimum Coherence0.0Hard-mask pixels with coherence below this threshold (no-data in output). Ignored when no matching coherence band is found. Leave at 0 and rely on coherence weighting (below) instead — hard masking can isolate regions and cause patchy 2π offsets.
Connectivity8Edge connectivity for the union-find: 4 = N/E/S/W only, 8 = also include diagonals.
Coherence-weighted reliabilitytrueWeight edge reliability by coherence: R = avg_coh / (D_a + D_b + eps). Makes the union-find walk along high-coherence corridors first, substantially reducing 2π patches in noisy scenes. Ignored when no matching coherence band is found.
Cycle-skip correctiontruePost-unwrap correction that snaps each pixel to the nearest integer 2π cycle relative to a local median. Specifically fixes the "patchy" Herraez artefact where a locally-correct group sits at the wrong cycle relative to its neighbours. Runs as an iterative multi-pass (3 → 5 → 9 → 15 → 25 → 41 → 71 → max), each pass cleaning patches up to its half-window radius and feeding the next. Only moves pixels by integer multiples of 2π, so real signal is preserved exactly.
Cycle-skip max window21Maximum window size (odd integer) for the multi-pass cycle-skip ladder. Pick the largest patch width you expect in the scene — 21 handles typical InSAR patches up to ~40 px wide; 41–71 for very noisy scenes; 101 for large-scale residual offsets. Cost grows ~quadratically in window size, so don't over-specify.
Median filter size0Optional post-unwrap median filter kernel size (0 = off, 3, 5, or 7). Cosmetic noise reduction applied after the cycle-skip correction. Smooths sub-cycle noise but slightly blurs real fringes; leave off unless visualisation requires it.

Recommended preprocessing

  • Apply a Goldstein phase filter to the wrapped interferogram first. It boosts SNR without altering fringe phase, and is the standard complement to any quality-guided unwrapper.
  • Make sure your input product has a matching coherence band (interferogram + coherence in the same product), so the coherence weighting and cycle-skip steps have something to work with.

References

  • Herráez M. A., Burton D. R., Lalor M. J., Gdeisat M. A. (2002). Fast two-dimensional phase-unwrapping algorithm based on sorting by reliability following a noncontinuous path. Applied Optics 41(35), 7437-7444.

Choosing between Herraez and snaphu

Use HerraezUse snaphu
Small/medium scenes, no native helper, batch graphs Large scenes, low coherence, dense fringes, residue-heavy interferograms - snaphu's Lp-norm MCF solver handles these better.

Split Stack into Multiple References

This menu item runs the MultiMasterStackGenerator graph, which takes a single coregistered SLC stack (one reference acquisition + N−1 secondary acquisitions, as produced by Coregistration or DEM-Assisted Coregistration) and splits it into a set of products, one per acquisition acting as the reference. The output is the input required for multi-reference / multi-master InSAR processing where every interferogram in a small-baseline network is formed against the closest acquisition in time and space rather than against a single global master.

Input

A single-master coregistered SLC stack — i.e. a SNAP product whose bands are grouped by master / slave date with the standard i_<date> / q_<date> naming convention.

Output

A set of products, one per acquisition designated as reference. Each output product is a stripped-down stack containing the chosen reference and the N−1 other acquisitions re-cast as secondaries against it. The metadata for each output is rewritten so that the master entry in the abstract metadata reflects the new reference.

Typical workflow

  1. Coregister all SLCs into one stack (e.g. DEM-Assisted Coregistration or the TOPS Back-Geocoding + Enhanced Spectral Diversity chain).
  2. Run Split stack into multiple references on that coregistered stack.
  3. For each output product, use the InSAR network design tools (InSAR Stack Overview, Multi-Reference InSAR) to form the interferograms appropriate to that reference.
  4. Combine the results downstream (e.g. via SBAS-style network inversion in StaMPS / MintPy after export).

Notes

  • This operator does not re-coregister: the underlying samples are the same as in the input stack. Only the reference / secondary roles are reassigned.
  • If the input stack was coregistered with non-trivial enhanced spectral diversity (ESD) residual offsets that depended on the original master, those offsets are baked in; re-running ESD against the new reference will give residuals close to zero only if the original ESD converged.

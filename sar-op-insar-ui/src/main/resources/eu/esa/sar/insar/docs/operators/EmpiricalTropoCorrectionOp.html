Empirical Tropospheric Correction

Empirical Tropospheric Correction Operator

Stratified tropospheric delay is, to first order, a function of topographic elevation: water vapour concentration and temperature both decrease with height, so SAR signals from higher ground accumulate less excess path delay. This operator estimates the topography-correlated component of the tropospheric phase per unwrapped interferogram by least-squares fit to a DEM, then subtracts the modelled phase from every pixel.

The model is the standard "phase-elevation regression" used in MintPy (tropo_phase_elevation) and GIAnT, and is the empirical fallback when external models such as GACOS or ERA5 ray-tracing are not available [1][2]:

  • Linear:   phase(z) = c0 + c1·z
  • Quadratic:   phase(z) = c0 + c1·z + c2·z2

Coefficients are fit globally per phase band on a coherence-masked sample grid drawn at a configurable stride; the same coefficients are then applied per-pixel using DEM elevation lookup. The operator removes only the elevation-correlated component of the delay — turbulent residuals and unstratified tropospheric noise pass through untouched. For strong stratification (large relief, humid lower atmosphere) it typically removes 30–80 % of the tropospheric phase variance.

Input

A SNAP product containing one or more unwrapped interferometric phase bands (Unit = "phase"). Typical upstream chain:

  • Coregistration (Back-Geocoding + ESD for TOPS, or DEM-Assisted Coregistration for stripmap).
  • Interferogram formation with topographic phase removal enabled.
  • Multilook + Goldstein phase filtering.
  • Phase unwrapping with SNAPHU (export → external snaphu → import).

The operator auto-pairs each phase band with the matching coherence band (matched by master/slave date suffix). When a coherence band is available, low-coherence samples are excluded from the regression sample population.

Output

A new product mirroring the source structure, with each unwrapped phase band replaced by its corrected counterpart:

  • Corrected phase bands (Unit = "phase"), in the same place as the input phase bands.
  • All non-phase source bands (coherence, intensity, virtual phase, elevation) are passed through unchanged.

The fitted coefficients are written to the SNAP log so the user can audit the magnitude of the removed phase ramp.

Parameters

  • Digital Elevation Model: name of the DEM used to look up per-pixel elevation. Default: Copernicus 30m Global DEM. Any DEM registered in SNAP is accepted.
  • DEM Resampling Method: resampling kernel used by the elevation model when interpolating between DEM posts. Default: bilinear.
  • External DEM: optional path to a local DEM file used in place of a registered DEM (must contain a valid geocoding).
  • DEM No Data Value: value treated as no-data for an external DEM.
  • Regression model: linear (two coefficients) or quadratic (three coefficients). Quadratic is helpful when the relief spans a large vertical range and the troposphere is not well approximated by a linear vertical gradient; for most Sentinel-1 scenes the linear model is sufficient.
  • Sample stride: stride (in pixels) of the regression sample grid. Smaller values fit on more samples (slower, possibly more robust to local outliers); larger values are faster but weight the fit toward broad scene-scale trends. Default: 20.
  • Coherence threshold: samples with coherence below this value are excluded from the regression. Has no effect when no coherence band is paired with the phase band. Default: 0.3.
  • Minimum elevation (m): pixels whose DEM elevation is at or below this floor are excluded from the fit (and left unchanged on output). Useful to drop ocean / sea-level no-data samples that otherwise flatten the slope. Default: 1.0.

Notes and limitations

  • The operator works on unwrapped phase. Run it after SNAPHU import; running it on wrapped phase yields meaningless results.
  • It removes only the topography-correlated component. For production deformation studies, follow it with a turbulent / common-mode atmospheric removal step (e.g., temporal high-pass on a SBAS time series, or an external GACOS/ERA5 correction).
  • If the network is sparsely sampled or the relief is small, the regression may be ill-conditioned and the operator will log a warning and apply zero correction (passing the phase through unchanged) rather than fabricating a noisy slope.
  • Pixels with no DEM coverage (DEM no-data) are passed through unchanged.

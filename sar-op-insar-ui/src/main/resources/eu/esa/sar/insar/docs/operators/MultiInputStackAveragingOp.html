Multi-Input Stack Averaging

Multi-Input Stack Averaging Operator

Aggregates N independent co-located products (i.e. same raster width / height) into a single image carrying one of the following descriptive statistics, computed pixel-wise across the input set:

  • Mean Average
  • Minimum
  • Maximum
  • Standard Deviation — population definition, sqrt(mean(x2) - mean(x)2)
  • Coefficient of Variation — SAR-physical definition, sqrt(mean(x4) - mean(x2)2) / mean(x2). This is the CV of intensity, not of amplitude. If you need amplitude-CV, divide stddev by mean outside the operator.

Complementary to Stack Averaging, which reduces over the bands of a single coregistered stack product. Use this operator when your inputs are separate products that have not been combined into a SNAP stack.

Input

  • Two or more products, all sharing the same scene raster width and height.
  • Band names are matched by stripping date suffixes (via StackUtils.getBandNameWithoutDate), so e.g. Sigma0_VV_01Jan2025 in product 1 and Sigma0_VV_13Jan2025 in product 2 are aggregated together into Sigma0_VV.

Output

  • One virtual band per band-name group, named after the stripped band name, carrying the configured statistic expression. The expression is evaluated lazily by the SNAP BandMaths engine, so memory use is bounded by the tile size, not by the stack length.
  • AbstractMetadata.coregistered_stack is set to 0 in the target metadata.

Parameters

ParameterDefaultValues
StatisticMean Average Mean Average, Minimum, Maximum, Standard Deviation, Coefficient of Variation

Notes

  • The Coefficient of Variation formula is intentional — it treats x as a SAR amplitude and computes the CV of the corresponding intensity. Cross-check before using on non-SAR data.
  • Inputs must be pre-coregistered to a common geometry; this operator does not perform geometric alignment. Use Coregistration + Create Stack upstream if needed.

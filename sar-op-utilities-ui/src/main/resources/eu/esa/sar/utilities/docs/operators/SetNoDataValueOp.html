Set No-Data Value

Set No-Data Value Operator

Sets the no-data value and the NoDataValueUsed flag on every band of the source product. Useful when an upstream operator has produced bands whose sentinel-value flag is missing or wrong, causing downstream operators (statistics, masking, terrain correction) to treat the sentinel as a valid measurement.

Input

Any product. Virtual bands and physical bands are both handled.

Output

A new product mirroring the source structure. Every band is copied through with its no-data value and no-data-used flag set to the user-specified values. Pixel samples are not modified.

Parameters

  • No Data Value Used — whether the no-data value should be honoured by downstream operators. Default: true.
  • No Data Value — the sentinel value treated as no-data. Default: 0.0. Common alternatives: NaN for floating-point bands, -9999 for integer-typed elevation / classification rasters.

Notes

  • This operator only changes metadata flags — it does not edit pixels. To replace actual sample values with a sentinel, use the BandMaths operator (e.g., if x < 0 then NaN else x).
  • If the no-data value is NaN, set it via the operator parameter as NaN; SNAP recognises the literal.

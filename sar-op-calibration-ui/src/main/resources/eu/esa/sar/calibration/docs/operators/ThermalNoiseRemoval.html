Thermal Noise Removal

Thermal Noise Removal Operator

Thermal noise correction can be applied to Sentinel-1 Level-1 SLC products as well as Level-1 GRD products which have not already been corrected. The operator can also remove this correction based on the product annotations (i.e. to re-introduce the noise signal that was removed). Product annotations will be updated accordingly to allow for re-application of the correction.

Level-1 products provide a noise LUT for each measurement data set. The values in the de-noise LUT, provided in linear power, can be used to derive calibrated noise profiles matching the calibrated GRD data.

Bi-linear interpolation is used for any pixels that fall between points in the LUT.

Parameters

  • Remove Thermal Noise – Subtract the thermal noise LUT from the pixel intensity.
  • Output Noise – Output the noise estimate as an additional band.
  • Re-Introduce Thermal Noise – Add back the thermal noise that was previously removed.
  • Clip Negative Values – When enabled (default), negative values resulting from the noise subtraction are replaced by a small positive floor value. When disabled, negative values are preserved in the output. See the guidance below on when to use each option.

Clipping of Negative Values

Subtracting the estimated thermal noise can produce negative intensity values. This happens because the noise annotation provides a low-frequency estimate of the actual thermal noise, and pixel-level noise may deviate above or below that estimate.

Whether to clip these negative values depends on how the denoised data will be used downstream:

  • Without spatial or temporal averaging (e.g. object detection, thresholding, direct analysis): Enable clipping (the default). Negative intensity values have no physical meaning and may cause issues in subsequent processing steps.
  • With spatial or temporal averaging (e.g. multi-looking, temporal composites, speckle filtering): Disable clipping. Since pixel-level noise is equally likely to be above or below the estimated level, clipping before averaging introduces a positive bias. In this case, keep the negative values during averaging, and optionally clip after the averaging step if desired.

For further details refer to Section 6.2 of the ESA Technical Note MPC-0392 DI-MPC-TN v2.2, "Thermal Denoising of Products Generated by the S-1 IPF" (2024).

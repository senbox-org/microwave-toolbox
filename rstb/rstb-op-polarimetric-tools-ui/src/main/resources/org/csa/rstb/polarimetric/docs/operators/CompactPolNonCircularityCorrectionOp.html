Compact-Pol Non-Circularity Correction Operator

Compensates the residual H/V transmit phase imbalance δ on a compact-pol (LCH/RCH) acquisition. Real-hardware compact-pol radars do not transmit a perfectly circular wave; the small phase imbalance between the H and V transmit chains makes the actual transmit polarization slightly elliptical. If ignored, this bias propagates into every downstream Stokes-vector / m-χ / m-δ / H-α / CP-RVI / CP-decomposition product. The effect is most visible on low-backscatter targets where the bias dominates the signal (sea ice, calm water, dark vegetation).

RCM CP SLC products carry a per-acquisition calibrated δ. CEOS-ARD CP products have this correction already baked in - SNAP-on-Level-1 does not.

Algorithm

The correction is a phase rotation applied to the C2 covariance cross-term:

C12_corrected = C12_observed · exp(j·δ)
              = (C12_real cos δ - C12_imag sin δ)
              + j(C12_real sin δ + C12_imag cos δ)

Intensities C11, C22 are invariant and pass through unchanged.

Input

  • Compact-pol C2 covariance product (output of Polarimetric Matrices in C2 mode on an LCH/RCH SLC). Raw LCH/RCH SLC input is rejected with a guidance message - build C2 first.

Output

  • Same C2 product with corrected C12_real / C12_imag bands. transmit_phase_imbalance_deg recorded in abstracted metadata.

Parameters

ParameterDefaultNotes
Transmit Phase Imbalance (deg)0.0 Scene-constant δ in degrees. Typical RCM values are a few degrees. Default 0 is a no-op so this operator is safe to leave in a graph for non-RCM CP data.

v2 upgrade path

Future versions will auto-read δ from RCM product metadata (calibration LUT in the manifest) so the operator becomes parameter-free on RCM input while keeping the manual override for other CP sensors.

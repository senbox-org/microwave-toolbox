Polarimetric Calibration (Quegan / Ainsworth)

Read this before running the operator

This operator is NOT the same as radiometric calibration. Most polsar workflows do not need it. Running it on data that is already polarimetrically calibrated will degrade the calibration, not improve it.

What this operator does

It estimates and corrects two kinds of distortion in the measured 2×2 scattering matrix:

  • Channel imbalance — complex amplitude/phase mismatch between the H and V transmit/receive paths.
  • Crosstalk — leakage between the polarisation channels caused by imperfect antenna isolation, described by four complex parameters δ1..4.

The output is a corrected full-pol scattering-matrix product (8 bands: i/q for HH, HV, VH, VV).

What this operator is NOT

It is not a replacement for the standard Calibration operator (radiometric calibration to σ0 / γ0 / β0). Radiometric calibration scales DN values to physical backscatter using the LUT shipped with the product; polarimetric calibration corrects channel mixing in the scattering matrix. They address completely different distortions. Use the standard Calibration op for radiometric calibration, always.

When SHOULD you run it?

Only when you have evidence that the polsar data has not been polarimetrically calibrated by the provider. Concrete cases:

  • Airborne SAR research data such as UAVSAR, AIRSAR, F-SAR, RAMSES, E-SAR, Pi-SAR, processed from raw signal data without on-orbit calibration.
  • Custom or experimental quad-pol products where the documentation explicitly states that crosstalk / imbalance has not been corrected.
  • Validation studies where you want to re-derive the calibration parameters and compare them against the provider's values.

The following missions deliver polsar products that have already been polarimetrically calibrated by the data provider. Running this operator on them will over-correct and corrupt the data:

Inputs

  • Full quad-pol scattering matrix product (FULL): 8 bands — i/q components of SHH, SHV, SVH, SVV.

Parameters

  • Calibration method:
    • Quegan: closed-form first-iteration solution (Quegan 1994).
    • Ainsworth: iterative refinement of Quegan (Ainsworth, Ferro-Famil & Lee 2006). Default.
  • Ainsworth Iterations: number of refinement iterations (default 4). Convergence is typically achieved in 3–5 iterations.
  • Stats Sub-sampling: only every Nth pixel in row and column is used to estimate scene statistics. 10 (default) gives 1 % of the scene — sufficient for distributed-target statistics and 100× faster than full-scene sampling.
  • Save Distortion to Metadata: stores the estimated f and δ1..4 in the target product's abstracted metadata under Polarimetric_Calibration.

Outputs

  • Corrected scattering matrix in the same FULL band layout as the input.
  • (Optional) Distortion parameters in metadata: complex channel imbalance f and crosstalk δ1..4.

Validation expectation

This implementation follows the published descriptions of Quegan (1994) and Ainsworth (2006). Sign conventions and basis transformations differ between published implementations; users running quantitative polarimetric analyses should validate against PolSARpro or the original published reference data (e.g. AIRSAR San Francisco distortion calibrations) before adopting the results in production.

References

[1] Quegan, S., 1994. A unified algorithm for phase and crosstalk calibration of polarimetric data — theory and observations. IEEE TGRS 32(1), 89–99.

[2] Ainsworth, T.L., Ferro-Famil, L., Lee, J.S., 2006. Orientation angle preserving a posteriori polarimetric SAR calibration. IEEE TGRS 44(4), 994–1003.

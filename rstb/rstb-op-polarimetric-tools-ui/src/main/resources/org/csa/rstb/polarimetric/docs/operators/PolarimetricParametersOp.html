Polarimetric Parameters

Polarimetric Parameters Operator

Computes a configurable bank of general-purpose polarimetric scalar parameters and channel-algebra outputs from a quad-pol, dual-pol or compact-pol SAR product. The operator works on either the original complex scattering matrix or on the locally averaged coherency / covariance matrix produced by spatial multilooking inside a sliding window.

Input modes (auto-detected)

  • Intensity bands — the source product carries named intensity bands (_HH, _HV, _VV, _VH). Channel ratios and indices (RFDI, CSI, VSI, BMI, ITI) are derived directly from those bands.
  • Matrix elements — the source is a C2 / C3 / T3 / FULL covariance or coherency matrix product produced by Polarimetric Matrix Generation. Diagonal elements give the channel intensities; off-diagonal elements give the complex correlations used for the Phase and Coherence outputs.

For dual-pol C2 input the available channels depend on the polarisation pair encoded in the product (HH+HV, VV+VH or HH+VV); outputs that need missing channels are rejected at validation.

Available outputs

  • Span — total backscattered power, sum of |SHH|2, 2|SHV|2, |SVV|2 for quad-pol. Requires a complex C3/T3/FULL input.
  • Pedestal Height — ratio of the smallest to the largest eigenvalue of the coherency matrix; a measure of polarimetric noise / depolarisation floor. Requires a complex C3/T3/FULL input.
  • Radar Vegetation Index (RVI) — ratio of cross-pol to total power, sensitive to volume scattering from canopies.
  • Radar Forest Degradation Index (RFDI) — (HH−HV) / (HH+HV).
  • Canopy Structure Index (CSI), Volume Scattering Index (VSI), Biomass Index (BMI), Interaction Index (ITI) — structural indices from Pope et al. (1994).
  • Ratios: HH/VV, HH/HV, VV/VH (linear power).
  • Complex Correlation (matrix input only) — |cij| / √(cii·cjj) between channel pairs in [0, 1]. Output bands: Coh_HHVV, Coh_HHHV, Coh_VVVH as available per input mode.
  • Off-diagonal Phase (matrix input only) — arg(cij) in radians. Output bands: Phase_HHVV, Phase_HHHV, Phase_VVVH as available per input mode.

Notes on Phase and Coherence outputs

  • For C3 / T3 / FULL input these are the C3 lex-basis correlations — T3 input is converted to C3 internally so the user always sees HH/HV/VV-named outputs regardless of whether the matrix product was generated as T or C.
  • The C3 c22 element stores 2|HV|2, so |HV| amplitudes and HV-paired correlations are normalised by the √2 basis factor before being written.
  • For C2 dual-pol input only one phase/coherence is meaningful (between the two channels actually present); the corresponding Phase_/Coh_ band carries that pair.

Parameters

  • Use Mean Matrix — when true, Span / Pedestal / RVI are computed on the locally averaged coherency matrix (recommended). When false, they are computed on the per-pixel single-look matrix (noisier).
  • Window Size X / Y — sliding window for the mean matrix. Default 5×5.
  • Output flags — one boolean per output band.

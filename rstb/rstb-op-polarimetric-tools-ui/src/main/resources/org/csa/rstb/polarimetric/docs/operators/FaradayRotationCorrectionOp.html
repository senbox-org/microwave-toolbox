Faraday Rotation Correction

Faraday Rotation Correction Operator

Removes the ionospheric Faraday-rotation signature from a quad-polarimetric SAR product. As the radar wave traverses the ionosphere, the plane of polarization rotates by an angle Ω that depends on the total electron content (TEC), the geomagnetic field component along the line of sight, and the radar wavelength (Ω ∝ λ2). At L-band and longer this can be tens of degrees and badly distorts the measured polarimetric quantities; at C-band and X-band the effect is small and usually negligible.

For a quad-pol product the observed and true scattering matrices are related by:

Sobs = R(Ω) · Strue · R(Ω)

where R(Ω) is the 2×2 rotation matrix. This operator estimates Ω per pixel from the observed scattering matrix using the Bickel-Bates / Freeman estimator:

Ω = (1/4) · arg( (Z12 − Z21) · conj(Z22 + Z11) )

where Z is the circular-basis transform of S, then applies the inverse rotation R(−Ω) to recover Strue.

Input

Full quad-pol SLC product. The operator rejects dual-pol and compact-pol inputs.

Output

A quad-pol product with the Faraday rotation removed. Optionally, an additional band carries the per-pixel estimated rotation angle Ω (degrees).

Parameters

  • Window Size — sliding window size used to spatially average the Zij products before computing Ω. Default 10 (range 1–100). Larger windows produce a smoother Ω map at the cost of resolution.

Notes

  • The correction is most relevant for L-band (ALOS-1/2/4, SAOCOM, NISAR) and P-band (BIOMASS) data. Applying it to C-band or X-band is harmless but usually has negligible effect.
  • The Bickel-Bates estimator assumes reciprocity (SHV = SVH); cross-channel SNR correction (Cross-Channel SNR Correction) is recommended as a pre-step.
